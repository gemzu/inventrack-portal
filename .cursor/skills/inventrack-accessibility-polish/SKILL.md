# INVENTRACK Accessibility Polish

Use this skill for UI refinement passes and before final delivery.

## Required Standards

1. All interactive controls have clear focus-visible styles.
2. Color is never the only status indicator.
3. Form controls include meaningful labels/placeholders.
4. Dialogs and overlays keep clear close/cancel paths.
5. Empty and error states provide action guidance.

## Checklist

```text
Accessibility Checklist
- [ ] Keyboard focus is visible on all critical actions
- [ ] Button labels are explicit ("Save", "Create User", etc.)
- [ ] Select/input text remains readable in all themes
- [ ] Status badges include text + color
- [ ] Error messages are actionable
```

## Table and Form Rules

- Keep table headings consistent and readable.
- Avoid compressed click targets; keep controls at usable heights.
- Keep validation feedback near the failing field.
